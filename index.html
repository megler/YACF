YACF

Yet Another CSS Framework

Intro

YACF (pronounced "Yack-Siff") may sound like what your cat does when he has a hairball, but in reality, it's a flexible CSS only Framework. With YACF, the hard work of layout has been thought out for you. All you need to do is strategically place your classes and you're all set.

Want to customize? Not a problem!

With YACF you have a customization page for all your Sassy/CSS needs so you can change anything to make it more suitable to your needs!

Installing YACF

Back to Top

Installing YACF is easy! Here are your options:

  1. Download the latest release from Github or
  2. Clone our Git repository. git clone git://github.com/megler/YACF.git

If you just want to use YACF out of the box with no modifications, download the latest release up in step 1 and extract either yacf-style.css or yacf-style-min.css and call it in your header.

You place the stylesheet in your header like this:

link rel="stylesheet" href="/directory-to-style-folder/yacf-style.css"

If you would like to customize YACF via the built-in SASS, then you can install using either step 1 or 2 above and extracting the entire package. You will still place your stylesheet normally, but now you will have all of the Sass files and optional npm scripts for automation.

That will give you full access to all the YACF classes. That's it! You're ready to roll!

The Grid

Back to Top

The main skeleton of the page is laid out using CSS Grid. This allows you more control over grid placement and page appearance.

To keep the container flexible, the .grid-container only declares the grid itself and has some responsive addons. The border you see on this page as well as the width of the container are set with separate classes so you have maximum flexibility.

To place the grid, use this class:

div class="grid-container"

From there, you can lay out your columns with some classes that probably seem familiar:

div class="col-span-5"

You can span 1-6 columns using this method. If you want your colums to be responsive, check out Let's Get Responsive! below.

Switching to Flexbox

Back to Top

Sometimes, Flexbox just makes more sense. Like when you are laying out horizontal menus. Fear not. It's as easy as selecting a new class:

div class="u--flex-container"

Need something centered in a way only Flexbox can provide? Say no more! YACF has you covered!

div class="u--flex__center"

Controlling container widths

Back to Top

Container width is controlled with:

div class="grid--width"

The default width is 70% and goes to 100% at tablet and mobile breakpoints. You can change that to whatever you like in the _customize.scss partial. Leaving this class off makes your container width 100%.

Now that you have your grid under control, let's see what else we can do!

For example if you wanted to change a div background you could:

div class="bg__blue"

Padding and Margin

Back to Top

Padding and margin can be controlled either through class placement or the customization page we'll talk about soon. Here are the base classes you can use:

div class="u--padding or u--margin"

You can set padding and margin in a number of configurations. Here are your options:

  • div class="u--padding__left"
  • div class="u--padding__right"
  • div class="u--padding__top"
  • div class="u--padding__bottom"
  • div class="u--padding__vertical"
  • div class="u--padding__left"
  • div class="u--padding__none"

You can swap out the word padding with margin and control that as well. Also, if you need to add or remove padding at the mobile breakpoint (currently this feature is only available at the mobile breakpoint, but you could add additional breaks in the customize.scss partial), you can add -mob to the class like this:

div class="u--padding__left-mob"

Let's say you have a div where you want left padding at desktop and tablet but no padding when it's mobile sizes. Here's how that would look:

div class="u--padding__left u--padding__none-mob"

The -mob class will override the regular class when the breakpoint is hit.

Typography

Back to Top

To make spacing across the framework easy math, the base font-size is set to 62.5% which means 1rem = 10px.

Now, any spacing is as easy as moving the decimal point. For example:

18px = 1.8rem

20px = 2rem

5px = .5rem

Note this math will works in REMs and EMs not PX. As a result, you'll find layout changes are a breeze.

Type Scale

Back to Top

YACF utilizes a responsive type scale. Never heard of it? Here's a resource.

This type scale responds over 3 breakpoints. Desktop, Tablet, and Mobile. It uses 3 different typescales to achieve the result. Because the type scale uses CSS Custom Properties, it takes a little more finagling to make changes. In the _customize.scss partial, you'll make changes like this:

--browser-default-font: 1.8; // This is a default base font of 18px

--multiplier: 1.250; // This is the typescale multiplier you want to use.

So if you want to change the type scale from 1.250 to 1.3, you'd do this:

h1, h2, h3, h4, h5, h6, p, li { --multiplier: 1.2; }

That would change the scale of all headers, p tags and li tags.

Let's Get Responsive!

Back to Top

YACF has 3 breakpoints:

Desktop: (min-width: 960px) // $bp-lg

Tablet: (min-width: 720px) and (max-width: 960px) // $bp-md and $bp-lg

Mobile: (min-width: 150px) and (max-width: 720px) // 150px and $bp-md

They are controlled with the following respond-to mixin:

@include respond-to(mobile)

You can replace mobile with tablet or desktop

The point of having a responsive grid system is having the flexibility to change the column layout at breakpoints. What looks terrific as 3 columns on desktop may look terrible on tablet or mobile.

YACF makes this a snap. You can change column count at breakpoints like this:

div class = "col-span-3 col-span-4@m col-span-6@s"

That translates to: at desktop, span 3 colmns, resize to 4 at the medium (tablet) breakpoint and 6 columns at the small (mobile) breakpoint.

If you check out the Example Page you'll see this in action. The main body text goes from 2 columns that span 3 wide (col-span-3) to full width at the tablet and mobile breakpoints. This gives you the flexibility to have the layout look however you like on each screen size.

Customizing

Back to Top

While YACF does most of the work for you, you may want to customize some styles and make them your own. The easiest way to do that is on the _customize.scss partial page.

Use this page to override any default setting and set it to whatever you like.

YACF SASS is laid out using a variation of 7-1 folder architecture. That means the majority of styles are in partials that have (hopefully) obvious names. (eg. if you are looking for navigation styles, check the navigation partial). About the only place this is not the case is with any feature that have a color option (buttons, text, etc). For that, check the _functions.scss partial.

Please note:

The _customize.scss partial is positioned at the bottom of the main.scss import page. It needs to be there if you intend to change any CSS Custom Properties such as type scale. This is due to how CSS Custom Properties take advantage of the top/down nature of CSS in order to work properly. If you move the _customize partial to a different position in main.scss, your Custom Properties overrides may not work as expected.
